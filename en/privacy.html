6. Information Security Policy

This section details the information security measures we implement in practice as an integral part of our Privacy Policy.

6.1 Communication Encryption

All communication between the application and servers is conducted via encrypted protocol (HTTPS/TLS). The server sets security headers including HTTPS enforcement (HSTS), iframe embedding prevention (X-Frame-Options), and Content-Security-Policy, to reduce exposure to common browser-based attacks.

6.2 User Authentication

User authentication is managed through Supabase's authentication service, based on signed access tokens (JWT). Password policy requires at least 8 characters including a letter and a digit. User passwords are not stored in our database.

6.3 Access Control

Protected endpoints require a valid access token. The authentication middleware checks every request before it reaches business logic, ensuring users can only access data associated with their own account.

6.4 Data Storage and Protection

User data is stored in a managed PostgreSQL database (Supabase) on secure cloud infrastructure. Data is stored encrypted at rest. Files uploaded by users are stored in private (non-public) storage, accessible only via signed, time-limited URLs.

6.5 API Layer Protection

  • Rate limiting to reduce abuse attempts and unusual server load
  • Strict validation of all user input (Zod schemas), including email address and phone number validation
  • Prisma ORM with parameterized queries, preventing SQL injection attacks
  • Restricted CORS policy permitting requests only from authorized origins

6.6 Secrets and Key Management

API keys and system secrets are stored as environment variables and not within application code. Incoming requests from external services (webhooks) are authenticated using shared secrets and timing-safe comparison to prevent request forgery.

6.7 Third-Party Data Processors

We rely on leading third-party providers to deliver the service. Only the information required for the relevant feature is shared with each provider:

  • Supabase — authentication, database, and file storage
  • OpenAI — text processing and AI capabilities
  • HeyGen — video creation
  • JAKO — WhatsApp message delivery

6.8 Payments

Our system does not store or process credit card details on our servers. All payment processing is handled exclusively through a dedicated external payment provider.

6.9 Logging and Monitoring

All server requests are logged with a unique request ID, enabling identification and diagnosis of errors and anomalies. Error messages in the production environment do not expose internal system details to users.

6.10 Security Incident Reporting

In the event of a material security incident affecting user data, we will send notification by email and in-app system message within 24 hours of identification.
